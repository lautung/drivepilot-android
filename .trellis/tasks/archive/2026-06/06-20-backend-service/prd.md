# 增加后端服务

## Goal

为现有 PhoneCar Android 原型增加可由客户端调用的后端能力，使当前纯本地演示数据能够按已确认的业务范围从服务端读取和写入。

第一阶段建设面向原型验证的演示业务后端，不接入真实车辆或真实支付系统。

## Confirmed Facts

- 当前仓库是单模块 Android 应用，没有后端模块、网络层、数据库或网络权限。
- 客户端使用 Kotlin、Jetpack Compose、ViewModel、Kotlin Flow 与 Preferences DataStore。
- 当前全部数据与交互状态集中在 `DemoState`，通过 Vehicle、Content、Service、Profile 四类 Repository 操作。
- 现有功能包括车辆控制演示、发现内容、选车与服务预约、订阅、个人隐私设置。
- 当前应用明确不连接真实车辆，也不执行真实支付、导航、蓝牙、相机、定位或远程车控。
- README 已说明现有 Repository 接口可作为以后替换真实后端的数据边界，但当前接口只有本地实现。

## Requirements

- 第一阶段支持用户登录，并隔离不同用户的数据。
- 第一阶段支持模拟车辆状态与个人设置在 Android 客户端和服务端之间同步。
- 第一阶段支持创建和读取维保预约、车辆预订与软件订阅记录，但不执行真实支付。
- 第一阶段支持从服务端读取发现内容。
- 发现内容图片存储在用户搭建的 MinIO 中，不继续依赖 Android 包内素材。
- 后端提供受权限保护的图片上传、查询和删除管理能力，并保存对象键、访问地址、媒体类型、大小等元数据。
- 第一阶段不开发独立 Web 管理后台。
- 管理能力通过 `ADMIN` 角色保护的 REST API 提供，并通过 OpenAPI/Swagger 页面支持调试。
- 管理 API 同时覆盖发现内容的创建、查询、修改、发布/下架和删除。
- 首个管理员账号由 `ADMIN_USERNAME` 和 `ADMIN_PASSWORD` 环境变量初始化，仅在账号不存在时创建。
- 密码必须使用强单向哈希保存；普通注册只能创建 `USER`，不能由客户端指定或提升角色。
- 所有车辆控制操作只修改服务端模拟状态，不向真实车辆下发指令。
- 后端框架使用 Spring Boot。
- 后端使用 Java 21。
- 后端使用 PostgreSQL 持久化业务数据，并使用 Flyway 管理数据库迁移。
- 本地开发通过 Docker Compose 启动 PostgreSQL。
- Android 客户端新增用户名和密码注册、登录、退出流程。
- 认证使用短期 JWT 访问令牌和可刷新令牌，并隔离不同用户的数据。
- 第一阶段不接短信、邮箱验证或第三方登录。
- Android 客户端断网时可以读取最近一次成功同步的缓存。
- 车辆状态、个人设置、预约和订阅等写操作必须联网；失败时保留服务端已确认状态并显示可重试错误。
- 第一阶段不实现离线写入队列或客户端与服务端的冲突合并。
- 跨设备同步的服务端状态包括模拟车控、个人隐私设置和内容关注状态。
- 车辆预订、维保预约和软件订阅建模为独立服务端业务记录，不再使用单个布尔值表达。
- 发现页当前标签、哨兵摄像头查看角度、未提交的车型/日期选择和弹窗状态继续保留在 Android 本地。
- 用户首次登录后，以服务端默认业务状态覆盖对应的本地缓存；旧的匿名 DataStore 业务状态不自动上传。
- 第一阶段通过 Docker Compose 在本地一键启动 Spring Boot 与 PostgreSQL。
- 后端需要能够构建独立 Docker 镜像，Android 客户端可通过构建配置切换 API 基础地址。
- 第一阶段不绑定云厂商，不包含域名、HTTPS 证书或公网部署。
- Docker Compose 同时启动 MinIO 并初始化所需存储桶。
- MinIO endpoint、access key、secret key 和 bucket 名称均通过环境变量配置，可切换到用户独立部署的 MinIO。
- MinIO 存储桶保持私有；发现内容 API 返回短期有效的预签名读取 URL。
- 图片上传只允许 `ADMIN`，客户端和普通用户不得直接获得 MinIO 凭证或写权限。
- 后端与 Android 客户端之间需要定义稳定、可测试的接口契约。
- 需要明确哪些现有本地状态迁移到服务端，哪些继续留在设备本地。
- 引入网络失败后，需要定义加载、错误、重试与离线行为，不能让异常直接进入 Compose 页面。
- 不得把原型操作误实现为真实车辆控制或真实支付，除非后续明确扩展任务范围并提供对应外部系统。

## Acceptance Criteria

- [x] `docker compose up --build` 可启动 Spring Boot、PostgreSQL、MinIO 和存储桶初始化服务，健康检查通过且 Flyway 自动完成迁移。
- [x] 普通用户可完成用户名密码注册、登录、令牌刷新和退出；密码不以明文保存，刷新令牌可轮换和撤销。
- [x] 两个不同用户读取或修改模拟车辆状态、隐私设置、预约、订阅和关注数据时互不可见。
- [x] 模拟车辆状态和个人设置支持读取与修改，并通过版本字段防止跨设备静默覆盖。
- [x] 用户可创建并查询车辆预订与维保预约，可启用、停用并查询软件订阅；所有操作均不执行真实支付或真实车控。
- [x] 已发布的发现内容可分页读取，草稿或已下架内容不会出现在普通用户接口中。
- [x] `ADMIN` 可通过管理 API 上传、查询和删除图片，并创建、修改、发布、下架和删除发现内容；普通用户访问管理 API 返回 `403`。
- [x] 上传文件经过大小、格式和内容校验，使用随机对象键写入私有 MinIO 桶；被内容引用的媒体不能被直接删除。
- [x] 发现内容返回短期预签名图片 URL，Android 能加载并以媒体 ID 作为稳定缓存键。
- [x] Android 新增注册、登录和退出入口，首次登录以服务端状态覆盖同步字段，退出时清除用户令牌及用户级缓存。
- [x] Android 在线写入失败时恢复到最近一次服务端确认状态并显示可重试错误；离线时仍可读取最近缓存，但不能提交写操作。
- [x] Android 继续把页面标签、摄像头查看角度和未提交表单选择保存在本地，不上传这些 UI 状态。
- [x] 后端提供 `/v3/api-docs` 与 Swagger UI；API 错误使用统一 Problem Details 结构，日期、枚举、UUID 与分页格式有自动化契约测试。
- [x] 后端测试、Android JVM 测试、Android lint、Debug 构建和现有设备导航测试通过。

## Out of Scope Until Explicitly Approved

- 连接真实车辆或下发真实车控指令。
- 真实支付、地图导航、定位、相机、蓝牙和推送通知。
- 面向生产环境的高可用、合规审计和正式发布基础设施。
- 独立 Web 管理后台、密码找回、短信/邮箱验证、第三方登录和管理员在线授权界面。
- 离线写入队列、自动冲突合并、多车辆绑定和真实道路救援调度。
